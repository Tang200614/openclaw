# OpenClaw 多 Agent 协作提示词模板

## 使用方式

复制以下提示词模板，替换 `[任务描述]` 部分，然后执行：

```bash
openclaw agent -m "[粘贴提示词内容]"
```

---

## 模板：多 Agent 协作执行复杂任务

你是一个多 Agent 协作系统。我需要你协调完成以下复杂任务：

**任务：[在这里描述你的复杂任务]**

请按照以下步骤执行：

### Step 1: 任务分析与拆分
分析上述任务，将其拆分为 2-5 个独立的子任务。每个子任务应该：
- 有明确的目标和输出
- 尽可能独立，可并行执行
- 使用适当的工具（Read、Bash、Grep、Glob等）

列出子任务清单：
- 子任务1: [描述] → 预期输出
- 子任务2: [描述] → 预期输出
- ...

### Step 2: 并行执行子任务
使用并行工具调用同时执行所有子任务：
```
同时调用：
1. [工具调用1] - 执行子任务1
2. [工具调用2] - 执行子任务2
3. ...
```

### Step 3: 结果汇总
收集所有子任务的结果，进行以下处理：
1. 整合所有输出数据
2. 分析各结果之间的关系
3. 生成完整的最终答案
4. 提供结论和建议

### Step 4: 输出最终报告
按照以下格式输出：

```
=== 任务执行报告 ===

## 任务概述
[原始任务描述]

## 子任务执行情况
| 子任务 | 状态 | 关键结果 |
|--------|------|----------|
| 任务1 | ✅ | [摘要] |
| 任务2 | ✅ | [摘要] |
| ... | ... | ... |

## 综合分析
[跨子任务的分析和发现]

## 最终答案
[完整、结构化的答案]

## 结论与建议
[总结和行动建议]
```

请开始执行。

## 示例任务

### 示例1: 系统配置全面审查

```bash
openclaw agent -m "你是一个多 Agent 协作系统。我需要你协调完成以下复杂任务：

任务：全面审查 ~/.openclaw/ 目录的配置，识别潜在问题并提出优化建议

请按照以下步骤执行...
[复制上面的模板步骤]"
```

### 示例2: 代码库分析

```bash
openclaw agent -m "你是一个多 Agent 协作系统。分析当前代码库的：
1. 代码统计（文件数、行数）
2. 配置文件检查
3. 依赖关系分析
4. 潜在问题识别

并行执行以上分析任务，然后汇总生成完整报告。"
```

### 示例3: 多数据源整合

```bash
openclaw agent -m "你是一个多 Agent 协作系统。需要从以下数据源收集信息并整合：
1. 读取配置文件获取基础设置
2. 扫描目录结构了解项目组织
3. 分析日志文件了解运行情况
4. 检查模型配置了解可用资源

并行收集以上信息，然后生成配置总览报告。"
```

## 工作原理

这个方案通过**提示词工程**在单个 Agent 内模拟多 Agent 协作：

1. **Coordinator 角色** - 提示词中的任务分析和拆分步骤
2. **Workers 角色** - 提示词中的并行工具调用
3. **Aggregator 角色** - 提示词中的结果汇总步骤

虽然没有真正的多个 Agent 实例，但通过角色划分和步骤规划，实现了类似的效果。

## 优势

- ✅ 无需配置额外的 Agent
- ✅ 利用 main agent 即可实现
- ✅ 真正的并行执行（并行工具调用）
- ✅ 自动结果汇总
- ✅ 结构化输出

## 局限性

- 单 Agent 上下文限制（任务过多时会受影响）
- 不如真正的多 Agent 系统可扩展
- 无法持久化子任务状态
